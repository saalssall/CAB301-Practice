    Big-O Time Complexity Analysis:
    
     1. Cocktail Shaker Sort:
        Perform a bubble-sort-style pass from left to right, swapping adjacent out-of-order pairs, 
        which pushes the largest element to the end. Then perform a pass from right to left, 
        swapping adjacent out-of-order pairs, which pushes the smallest element to the beginning. 
        Repeat, shrinking the unsorted range after each pair of passes, until a full round makes no swaps.
    
        1. Identify the loop(s) that do comparisons.
        2. Write a worst-case bound on comparisons per loop.
            While loop governs two for loops
            First for loop has (end-start) comparisons
            Second for loop has (end-start) comparisons
          
        3. If there are nested loops or repeated passes, use a summation.
            Worst case is when swap happens all the way until the end
        
        4. Simplify algebraically to a polynomial/log form.
            (n-1) + (n-2) + (n-3) + (n-4) + ... + 2 + 1
        
        5. Convert to Big-O by discarding constants and lower-order terms
        (consistent with Big-O being an upper-bound up to constant factors).
            Big-O complexity is O(n^2)
        
        2. Jum-search:
            You are given a sorted array of integers in ascending order. Instead of checking every element, 
            choose a fixed step size and ‘jump’ forward by that step, checking only the last element of each block 
            until you either land in a block that could contain the key or you run past the end. Once the right block is identified, 
            scan linearly inside that block to either find the key or conclude it isn’t there. Use a step size that makes the 
            search fastest when the array has n items, which is the (floor of) the square root of n. 
            
            1. Identify the loop(s) that do comparisons.
                While loop and for loop
            
            2. Write a worst-case bound on comparisons per loop.
                 While loop: Ceiling(n/sqrt(n)) * key comparisons 
                 For loop: Floor(n/sqrt(n)) * key comparisons 
                  
            3. If there are nested loops or repeated passes, use a summation.
                  Total comparisons (worst-case): Ceiling (sqrt(n) + Floor(sqrt(n))
                    <= 2*sqrt(n) + 1
                
            4. Simplify algebraically to a polynomial/log form.
                Worst-case: 2*sqrt(n) + 3 or 2*n^(1/2) + 3
                
            5. Convert to Big-O by discarding constants and lower-order terms
                (consistent with Big-O being an upper-bound up to constant factors).
                    Big-O complexity is O(n^(1/2)) = O(sqrt(n))
